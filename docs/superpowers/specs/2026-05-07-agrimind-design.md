## 1. 问题定义

现有农业AI普遍采用"一图一标签"的被动分类模式，存在三大局限：

1. **单图信息不足**：一张照片往往不够确诊，真正的植保专家会翻看叶背、检查茎秆、对比健康植株
2. **诊断不可解释**：模型输出一个标签，用户不知道为什么，不敢信
3. **无法表达不确定性**：模型明明判断错了，照样输出99%置信度

AgriMind 将作物诊断从图像分类问题重新定义为 **多模态推理问题**，通过多智能体协作实现可解释、可追溯、可校准的诊断。

## 2. 项目简介

现有农业AI普遍采用"一图一标签"的被动分类模式，信息不足且不可解释。AgriMind将作物诊断重新定义为多模态推理问题，提出三项创新：（1）诊断辩论协议DDP——创新性地将多智能体辩论机制引入农业诊断，初诊、质疑、仲裁三个AI专家结构化辩论达成共识，提升准确率与可解释性；（2）主动视觉问诊AVD——AI主动引导用户拍摄叶背、茎秆等关键视角，多轮收集证据后再判断，模拟专家问诊流程；（3）AgriReason农业多模态推理评测基准，填补该领域推理评测空白。系统基于国产Qwen2.5-VL大模型微调，支持病害、虫害、缺素、长势四类任务，可视化呈现辩论推理全过程，输出含鉴别诊断与证据溯源的完整报告。

## 3. 核心创新

### 3.1 创新点一：诊断辩论协议（Diagnostic Debate Protocol, DDP）

#### 动机

医学领域已有 MedAgents（Tang et al., arXiv:2311.10537）证明多Agent辩论可提升诊断准确率。但在农业视觉诊断领域，该方法尚未被探索。

#### 设计

三个Agent角色，共享同一个微调后的VLM，通过不同的 system prompt 实现专业化分工：

| Agent | 角色 | 职责 |
|-------|------|------|
| Proposer（初诊专家） | 资深植物病理学家 | 基于图像和上下文给出最可能的诊断，列出支持证据 |
| Challenger（质疑专家） | 严谨的植保审核专家 | 独立分析同一图像，找出初诊漏洞，提出替代诊断并用证据反驳 |
| Arbiter（仲裁专家） | 诊断委员会主席 | 综合双方论据做最终裁定，证据不足时明确表达不确定性 |

#### 辩论流程（2轮）

```
Round 1 — 交锋:
  Proposer: 提出诊断 + 支持证据
  Challenger: 质疑 + 替代诊断 + 反面证据

Round 2 — 裁定:
  Proposer: 回应质疑，可修改诊断
  Arbiter: 综合裁定，输出最终诊断
```

#### 输出格式

```
✅ 最终诊断：[病害名称]（置信度：高/中/低）
   支持证据：[1] ... [2] ... [3] ...

❌ 排除诊断1：[病害名称]
   排除原因：...

❌ 排除诊断2：[病害名称]
   排除原因：...

⚠️ 不确定因素：（如有）
```

#### 为什么是2轮而不是更多

- 1轮不够：Challenger的质疑无法得到Proposer回应
- 2轮刚好：交锋+裁定，逻辑闭环
- 3轮以上：推理延迟增加但准确率提升边际递减，不值得

### 3.2 创新点二：主动视觉问诊（Active Visual Diagnosis, AVD）

#### 动机

真正的植保专家不会只看一张照片就下结论。他们会翻看叶背确认孢子堆颜色、检查茎秆是否有病斑蔓延、对比周围健康植株排除环境因素。现有所有农业AI都缺少这个"主动追问"的能力。

#### 工作机制

1. 用户上传第一张图片
2. VLM 分析后评估"信息充分度"：当前图片能支撑多高置信度的诊断？
3. 信息不足时，生成 **针对性追问**，指向具体的拍摄部位和目的：
   - "请拍叶片背面——我需要确认孢子堆的颜色来区分条锈和叶锈"
   - "请拍一张同株健康叶片——排除营养缺素的可能"
   - "请拍茎秆基部——检查是否有根腐蔓延的迹象"
4. 用户补充图片后，重新评估信息充分度
5. 信息充分（或用户表示无法提供更多）时，进入DDP辩论诊断

#### 实现方式

不需要额外模型。在VLM微调数据中加入"多轮诊断对话"样本。这些样本由大模型自动合成（见6.2节），不需要人工编写。

追问策略基于 **信息增益最大化**：模型评估当前候选诊断列表，判断"哪一种额外视角最能区分候选项"，生成对应的追问指令。

#### 退出条件

- 信息充分度达到阈值 → 进入诊断
- 用户回复"没有了"/"就这些" → 基于现有信息进入诊断，但在报告中标注"建议补充XX信息"
- 最多追问3轮 → 强制进入诊断

### 3.3 创新点三：AgriReason 评测基准

#### 动机

医学有 MedQA/PubMedQA 等标准化推理基准，农业领域在多模态推理评测方面存在空白。现有农业数据集（PlantVillage等）只有图片+标签，没有推理过程。

#### 数据集构成

约 500-800 条样本，每条包含：

```json
{
  "image": "图片路径",
  "context": "可选的环境上下文（地域、季节、近期天气等）",
  "question": "诊断问题",
  "ground_truth": "正确诊断",
  "reasoning_chain": ["观察→推断→排除→结论的完整链路"],
  "difficulty": "easy | medium | hard",
  "task_type": "disease | pest | nutrient | growth"
}
```

#### 任务类型分布

| 任务类型 | 比例 | 描述 |
|---------|------|------|
| 单一病害诊断 (disease) | 35% | 标准病害识别+推理 |
| 虫害识别 (pest) | 25% | 害虫种类判断+危害程度评估 |
| 鉴别诊断 (differential) | 20% | 区分外观相似的病害/缺素 |
| 营养诊断 (nutrient) | 10% | 从叶色/形态判断缺素症状 |
| 不确定性处理 (uncertainty) | 10% | 图片模糊/信息不足时的合理表达 |

#### 自动化生成流程（零人工）

1. 从 PlantVillage / PlantDoc / IP102 / Cassava / Pest24 下载 (图片, 标签) 对
2. 调用 Qwen2.5-VL-72B（通义千问 DashScope 免费额度 或 本地GPU推理）
3. Prompt: "你是资深植物病理学家，对这张图片进行诊断，给出完整推理链：观察特征→特征指向→排除项→结论"
4. 对同一张图跑 3 次，取 **自一致性（self-consistency）** 结果
5. 3次结果一致的 → 纳入数据集；不一致的 → 自动剔除

#### 评测指标

- 诊断准确率（Top-1 Accuracy）
- 推理链质量（与参考链的语义相似度，用BGE-M3计算）
- 不确定性校准（ECE, Expected Calibration Error）
- 鉴别诊断覆盖率（正确排除了多少错误候选）

### 4.2 数据流详细说明

```
用户上传图片
  │
  ▼

  │
  ├─ 不足 → 生成追问指令 → 返回前端 → 用户补充图片 → 循环
  │
  └─ 充分 → 进入DDP辩论引擎
              │
              ├─ Proposer: 分析全部图片 → 输出诊断+证据
              │
              ├─ Challenger: 独立分析 → 输出质疑+替代诊断
              │
              ├─ Proposer: 回应质疑
              │
              └─ Arbiter: 综合裁定 → 最终诊断 + 推理链
                           │
                           ▼
                    输出生成模块
                     │  │  │
                     │  │  └─ RAG检索植保知识 → 处置建议

                              │
                              ▼
                        前端渲染完整报告
```

### 4.3 模块间接口

#### AVD → DDP 接口

```python
@dataclass
class DiagnosisContext:
    images: list[Image]          # 用户提供的所有图片（1-4张）
    image_descriptions: list[str] # VLM对每张图的结构化描述
    user_context: str | None      # 用户输入的文字上下文
    conversation_history: list    # AVD多轮对话历史
```

#### DDP → 输出生成 接口

```python
@dataclass
class DebateResult:
    final_diagnosis: str          # 最终诊断
    confidence: str               # "high" | "medium" | "low"
    supporting_evidence: list[str]
    rejected_diagnoses: list[dict] # [{name, reason}]
    uncertainty_notes: list[str]
    debate_transcript: list[dict]  # 完整辩论记录
    grounding_boxes: list[dict]    # 病灶标注坐标 [{x,y,w,h,label}]
```

## 5. 技术选型

### 5.1 多模态基座模型

**选择：Qwen2.5-VL-7B**

理由：
- 国产模型（竞赛评审加分项）
- 原生支持 Visual Grounding（输出bounding box坐标）
- 7B参数量在单卡GPU上可完成LoRA微调
- 中文能力强，适配中文农业文献
- 开源免费（Apache 2.0）

### 5.2 微调方案

- 方法：LoRA (rank=16, alpha=32)
- 框架：PEFT + Transformers + DeepSpeed ZeRO-2
- 训练数据：约 5000-8000 条指令微调样本（见6.2节）
- 训练时长预估：单卡 A100 约 8-12 小时，单卡 4090 约 16-20 小时
- 显存需求：约 20-24GB（7B模型 + LoRA，4090可运行）

### 5.3 RAG 知识检索

- 向量数据库：ChromaDB（本地运行，零成本）
- 嵌入模型：BGE-M3（多语言，开源）
- 知识源：
  - 《中国农作物病虫害图鉴》电子版
  - 植保技术手册 / 农药使用指南
  - 农业院校公开课程资料
- 检索策略：Top-5 相关段落，拼接到 Agent prompt 中

### 5.4 前端技术

- 框架：React 18 + TypeScript
- 样式：TailwindCSS
- 动画：Framer Motion（辩论气泡动画、标注渐现效果）
- 实时通信：WebSocket（辩论过程逐条推送到前端）
- 图表：Recharts（消融实验图表、性能雷达图）

### 5.5 后端技术

- 框架：FastAPI
- 实时推送：WebSocket endpoint
- 图片处理：Pillow + OpenCV（标注渲染）
- 模型推理：vLLM 或 Transformers pipeline
- 部署：Docker Compose（后端 + 模型推理 + ChromaDB）

## 6. 数据方案

### 6.1 训练数据来源（全部免费公开）

| 数据集 | 规模 | 内容 | 用途 |
|--------|------|------|------|
| PlantVillage | 54,306 张 | 38种作物病害 | VLM微调主数据 |
| PlantDoc | 2,598 张 | 真实田间拍摄 | 增强泛化能力 |
| IP102 | 75,000+ 张 | 102种害虫 | 虫害识别训练 |
| Cassava Disease | 21,397 张 | 木薯病害 | 补充多样性 |
| Pest24 | 25,000+ 张 | 24种常见害虫 | 补充虫害数据 |

### 6.2 微调指令数据生成（全自动）

使用 Qwen2.5-VL-72B 自动生成三类指令数据：

**类型一：单轮诊断 + 推理链（约 3000 条）**
```
Input: [图片] + "请诊断这张图片中的作物病害"
Output: "观察到...特征为...判断为...排除了...因为..."
```

**类型二：多轮问诊对话（约 1500 条）**
```
Turn 1 Input: [图片] + "这棵植物怎么了？"
Turn 1 Output: "我看到叶片有异常，但需要更多信息。请拍叶片背面。"
Turn 2 Input: [叶背图片]
Turn 2 Output: "确认看到X特征。最终诊断为..."
```

**类型三：辩论对话（约 1500 条）**
```
Proposer: "根据图像特征，我判断为X..."
Challenger: "但我注意到Y特征与X不符，可能是Z..."
Arbiter: "综合双方意见，最终判定为..."
```

生成方式：编写 Python 脚本，批量调用大模型生成。一次跑完，不需要人工参与。

### 6.3 数据质量保证（自动化）

- **自一致性过滤**：同一样本生成3次，结果一致的保留，不一致的剔除
- **格式校验**：脚本自动检查输出是否符合预定义的JSON schema
- **去重**：基于图像哈希 + 文本相似度去除重复样本

## 7. 消融实验设计

在 AgriReason Benchmark 上进行严格的对比实验：

| 编号 | 方法 | 描述 |
|------|------|------|
| B1 | ResNet-50 | 传统图像分类基线 |
| B2 | Qwen2.5-VL (zero-shot) | 不微调直接推理 |
| B3 | Qwen2.5-VL + CoT | 零样本 + Chain-of-Thought |
| B4 | Qwen2.5-VL-FT | 微调后直接推理（无DDP） |
| **O1** | **Qwen2.5-VL-FT + DDP** | 微调 + 辩论协议 |
| **O2** | **Qwen2.5-VL-FT + DDP + AVD** | 微调 + 辩论 + 主动问诊（完整系统） |

评估维度：
- Top-1 诊断准确率
- 推理链质量分数
- 不确定性校准误差（ECE）
- 平均推理延迟

预期趋势：B1 << B2 < B3 < B4 < O1 < O2
