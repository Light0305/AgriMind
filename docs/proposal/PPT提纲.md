## P2 痛点与问题（1分钟）

**标题**：现有农业AI——能用，但不好用

**左栏：三大核心痛点**

| 痛点 | 现状 | 后果 |
|------|------|------|
| 📷 **单图信息不足** | 一张照片就出结论 | 误诊率高——拍到的不一定是关键部位 |
| 🔒 **诊断不可解释** | 输出"锈病 96%"，没有理由 | 农户不敢信、不敢用药 |
| ⚠️ **无法表达不确定** | 明明判错了还显示99% | 错误诊断 → 错误用药 → 经济损失 |

**右栏：视觉对比图**

```
传统AI方式                      专家实际做法
┌──────────┐                  ┌──────────────────┐
│ 📷 一张图  │                  │ 📷 正面 + 背面     │
│    ↓      │                  │ 📷 茎秆 + 周围植株  │
│ 🏷️ 一个标签 │                  │    ↓              │
│ "锈病 96%" │                  │ 🧠 推理 + 排除      │
└──────────┘                  │    ↓              │
                              │ 📋 诊断 + 依据 + 建议│
                              └──────────────────┘
```

**底部一句话**：现有农业AI做的是"看图猜答案"，而真正的诊断需要"检查-推理-结论"。

## P3 我们的解决思路（45秒）

**标题**：重新定义问题——从图像分类到多模态推理

**核心重定义（页面中央，大字加粗）**：

```
图像分类问题 ────✕────→ 多模态推理问题

"上传一张图，输出一个标签"    "多角度采集证据，协作推理，
                             给出可解释的诊断报告"
```

**一句话定位**：让AI像专家团队一样"会诊"——不是一个人看一眼，而是多个专家反复讨论。

**三大创新模块预览（三列卡片）**：

| 创新点 | 名称 | 一句话 |
|--------|------|--------|
| 🔬 创新1 | **DDP 诊断辩论协议** | 三个AI专家辩论，不是投票 |
| 📷 创新2 | **AVD 主动视觉问诊** | AI主动追问，引导拍摄关键角度 |
| 📊 创新3 | **AgriReason 评测基准** | 首个农业推理评测数据集 |

## P4 创新点1 — 诊断辩论协议 DDP（1分30秒）

**标题**：DDP — Diagnostic Debate Protocol

**上半部分：三Agent角色图解（核心图示）**

```
         ┌──────────────┐
         │  🧑‍⚕️ 仲裁专家   │
         │   Arbiter     │
         │  "综合裁定"    │
         └──────┬───────┘
                │ 综合双方论据
       ┌────────┴────────┐
       ↓                 ↓
┌─────────────┐   ┌─────────────┐
│ 👨‍🔬 初诊专家   │   │ 🔍 质疑专家   │
│  Proposer   │   │  Challenger │
│ "我认为是…"  │←→│ "但你忽略了…" │
│  给出诊断    │   │  提出质疑    │
└─────────────┘   └─────────────┘
```

**中部：2轮辩论流程（时间线图）**

```
Round 1 ── 交锋 ──────────────────────
  Proposer  → "叶片出现橙黄色粉状物，诊断为小麦条锈病"
  Challenger → "但叶鞘无症状且温度偏高，可能是叶锈病"

Round 2 ── 裁定 ──────────────────────
  Proposer  → "补充：孢子堆排列成条状，非散生，坚持条锈病"
  Arbiter   → "采纳初诊。条状排列是条锈病关键区别特征。
              置信度：高 | 建议：确认叶背孢子颜色"
```

**下半部分：为什么是"辩论"而不是"投票"？**

| | 多数投票 | DDP辩论 |
|---|---------|---------|
| 输出 | 3票→结果 | 完整推理链 |
| 可解释性 | ❌ 无 | ✅ 每步有论据 |
| 错误可追溯 | ❌ | ✅ 可定位谁判错了 |
| 不确定性 | ❌ | ✅ Arbiter主动标注 |

**右下角：结构化输出示例**

```json
{
  "最终诊断": "小麦条锈病",
  "置信度": "高 ✅",
  "支持证据": ["橙黄色孢子堆", "条状排列"],
  "排除诊断": "叶锈病 ❌ — 孢子散生不符",
  "不确定因素": "⚠️ 建议拍摄叶背确认"
}
```

## P5 创新点2 — 主动视觉问诊 AVD（1分钟）

**标题**：AVD — Active Visual Diagnosis

**上半部分：对比图（左右双栏，核心图示）**

```
❌ 现有方式：被动分类              ✅ AVD：主动问诊
┌──────────────────┐          ┌──────────────────────┐
│ 用户上传 1 张图     │          │ 用户上传 1 张图         │
│       ↓            │          │       ↓                │
│ 模型立刻出结果       │          │ AI分析信息缺口          │
│ "锈病 96%"         │          │       ↓                │
│                    │          │ 🗣️ "请拍叶片背面         │
│ 信息不足？不管。     │          │    ——我需要确认孢子堆    │
│                    │          │      的颜色和排列方式"    │
└──────────────────┘          │       ↓                │
                              │ 用户补拍 → AI再分析       │
                              │       ↓                │
                              │ 📋 综合多张图给出诊断     │
                              └──────────────────────┘
```

**中部：追问策略——信息增益最大化**

```
  已上传：叶片正面 ──→ AI判断："可能是条锈病或叶锈病"
                        信息缺口：孢子排列方式未知
                            ↓
  AI追问："请拍叶片背面" ──→ 信息增益最大的下一步
                            ↓
  已上传：叶片正面 + 背面 ──→ AI判断："孢子条状排列，确认条锈病"
                             置信度从 中⚠️ → 高✅
```

**底部要点**：
- 每次追问都是经过信息增益计算的——不盲目追问，只问最关键的
- 最多追问2轮，控制用户负担
- 模拟植保专家田间问诊的真实流程

## P6 创新点3 — AgriReason 评测基准（45秒）

**标题**：AgriReason — 首个农业多模态推理评测基准

**上半部分：与现有数据集的对比表**

| 数据集 | 标签 | 推理过程 | 鉴别诊断 | 不确定性 |
|--------|------|----------|----------|----------|
| PlantVillage | ✅ 有 | ❌ 无 | ❌ 无 | ❌ 无 |
| PlantDoc | ✅ 有 | ❌ 无 | ❌ 无 | ❌ 无 |
| **AgriReason** | ✅ 有 | ✅ **完整推理链** | ✅ **排除诊断** | ✅ **置信校准** |

**一句话**：现有数据集只能回答"是什么"，AgriReason还能评估"为什么"和"有多确定"。

**中部：数据构成**

```
规模：500-800 条推理样本

5种任务类型：
  ① 病害识别与推理 ——"这是什么病？为什么？"
  ② 虫害识别与推理
  ③ 营养缺素诊断
  ④ 鉴别诊断     ——"A和B很像，如何区分？"
  ⑤ 不确定性表达  ——"信息不够时该怎么说？"
```

**底部亮点**：

```
生成方式：VLM引导 + 规则校验，全自动生成，零人工标注
数据质量：自动一致性检验 + 交叉验证
开源计划：项目完成后开源，填补领域空白
```

## 附：答辩Q&A预备

**高概率问题**：

1. **"你们和普通集成学习/多数投票有什么区别？"**
   → DDP辩论产生完整推理链和证据对抗，投票只有数字。辩论的关键产出是可解释性和不确定性表达，不仅仅是准确率提升。

2. **"为什么是两轮辩论，不是三轮、五轮？"**
   → 一轮不够充分（质疑无回应），两轮形成"交锋+裁定"逻辑闭环，三轮以上延迟显著增加但准确率边际递减。

3. **"7B模型够用吗？为什么不用更大的？"**
   → 7B经QLoRA微调后在专业领域足够强；更大模型推理成本成倍增加但DDP场景下需要3次推理，效率很关键；且单卡4090就能部署，落地性强。

4. **"AgriReason数据集怎么保证质量？"**
   → VLM引导生成 + 多维度自动校验（格式/一致性/覆盖度），交叉验证淘汰低质量样本，虽是自动生成但有系统化质控流程。

5. **"AVD追问会不会让用户体验变差？"**
   → 最多追问2轮，每次追问都基于信息增益计算（只问最关键的），且追问时会告诉用户原因。就像医生问诊——多问几句是为了更准。

6. **"项目进度风险怎么控制？"**
   → Phase 0已完成概念验证Demo，最高风险的技术可行性已初步验证；三人并行开发（模型/后端/前端），关键路径在模型微调（Phase 1），有明确milestone把控。
