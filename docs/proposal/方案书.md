## 二、项目背景与问题分析

农业是国民经济的基础产业，作物病虫害的精准诊断直接关系到粮食安全与农业可持续发展。随着人工智能技术的快速发展，基于深度学习的作物病害识别方法已取得显著进展，但现有农业AI系统普遍采用"一图一标签"的被动分类模式，在实际应用中暴露出三大核心局限：

**1. 单图信息不足。** 一张照片往往不够确诊。真正的植保专家在进行田间诊断时，会翻看叶背确认孢子堆颜色、检查茎秆是否有病斑蔓延、对比周围健康植株排除环境因素。现有AI系统缺少这种"主动追问"的能力，仅凭单张图片做出判断，误诊率居高不下。

**2. 诊断不可解释。** 模型输出一个标签，用户不知道判断依据是什么，难以建立信任。对于农业生产者而言，不可解释的诊断结论意味着不敢采信、不敢用药，AI的实际价值大打折扣。

**3. 无法表达不确定性。** 模型即使判断错误，依然输出高置信度分数，缺乏对自身推理可靠性的校准能力。这在实际场景中可能导致用户对错误诊断深信不疑，造成不可挽回的经济损失。

基于以上分析，AgriMind 将作物诊断从图像分类问题重新定义为**多模态推理问题**，通过多智能体协作实现可解释、可追溯、可校准的智能诊断。

## 三、项目简介

现有农业AI普遍采用"一图一标签"的被动分类模式，信息不足且不可解释。AgriMind将作物诊断重新定义为多模态推理问题，提出三项创新：（1）诊断辩论协议DDP——创新性地将多智能体辩论机制引入农业诊断，初诊、质疑、仲裁三个AI专家结构化辩论达成共识，提升准确率与可解释性；（2）主动视觉问诊AVD——AI主动引导用户拍摄叶背、茎秆等关键视角，多轮收集证据后再判断，模拟专家问诊流程；（3）AgriReason农业多模态推理评测基准，填补该领域推理评测空白。系统基于国产Qwen2.5-VL大模型微调，支持病害、虫害、缺素、长势四类任务，可视化呈现辩论推理全过程，输出含鉴别诊断与证据溯源的完整报告。

## 四、核心创新点

### 4.1 创新点一：诊断辩论协议（Diagnostic Debate Protocol, DDP）

#### 4.1.1 创新动机

医学领域已有研究（MedAgents, Tang et al., arXiv:2311.10537）证明多Agent辩论机制可有效提升诊断准确率。然而在农业视觉诊断领域，该方法尚未被探索和应用。AgriMind 创新性地将多智能体辩论机制引入农业诊断场景，构建结构化的诊断推理过程。

#### 4.1.2 协议设计

DDP 协议设计三个Agent角色，共享同一个微调后的视觉语言模型（VLM），通过差异化的系统提示词（System Prompt）实现专业化分工：

| Agent | 角色定位 | 核心职责 |
|-------|---------|---------|
| Proposer（初诊专家） | 资深植物病理学家 | 基于图像和上下文给出最可能的诊断，列出支持证据 |
| Challenger（质疑专家） | 严谨的植保审核专家 | 独立分析同一图像，找出初诊漏洞，提出替代诊断并用证据反驳 |
| Arbiter（仲裁专家） | 诊断委员会主席 | 综合双方论据做最终裁定，证据不足时明确表达不确定性 |

#### 4.1.3 辩论流程

辩论分为两轮进行，形成"交锋-裁定"的完整逻辑闭环：

```
Round 1 — 交锋:
  Proposer  → 提出诊断 + 支持证据
  Challenger → 质疑 + 替代诊断 + 反面证据

Round 2 — 裁定:
  Proposer  → 回应质疑，可修改或坚持原诊断
  Arbiter   → 综合双方论据，输出最终裁定
```

两轮设计的合理性论证：一轮不够充分，Challenger的质疑无法得到Proposer回应，论证不完整；两轮恰好形成"交锋+裁定"的逻辑闭环；三轮以上推理延迟显著增加但准确率提升呈边际递减，性价比不足。

#### 4.1.4 输出规范

DDP 的最终输出包含完整的鉴别诊断信息和证据溯源：

- **最终诊断**：病害名称 + 置信度等级（高/中/低）+ 支持证据列表
- **排除诊断**：被排除的候选诊断及排除原因
- **不确定因素**：信息不足或存疑之处的明确标注

### 4.2 创新点二：主动视觉问诊（Active Visual Diagnosis, AVD）

#### 4.2.1 创新动机

真正的植保专家在田间诊断时，不会只看一张照片就下结论。他们会翻看叶背确认孢子堆颜色、检查茎秆是否有病斑蔓延、对比周围健康植株排除环境因素。现有所有农业AI系统都缺少这种"主动追问"的能力。AVD 模块通过模拟专家问诊流程，使AI具备多轮主动收集诊断证据的能力。

#### 4.2.2 工作机制

1. 用户上传第一张图片。
2. VLM 分析后评估"信息充分度"——当前图片能支撑多高置信度的诊断。
3. 信息不足时，系统生成**针对性追问指令**，指向具体的拍摄部位和诊断目的。例如：
   - "请拍叶片背面——我需要确认孢子堆的颜色来区分条锈和叶锈"
   - "请拍一张同株健康叶片——排除营养缺素的可能"
   - "请拍茎秆基部——检查是否有根腐蔓延的迹象"
4. 用户补充图片后，系统重新评估信息充分度。
5. 信息充分（或达到最大追问轮次、用户明确表示无法提供更多）时，进入DDP辩论诊断。

#### 4.2.3 核心策略

追问策略基于**信息增益最大化**原则：模型评估当前候选诊断列表，判断"哪一种额外视角最能区分候选项"，生成对应的追问指令。退出条件包括三种情形：信息充分度达到阈值、用户主动结束、或已完成三轮追问。

#### 4.2.4 实现方式

AVD 不需要额外模型。通过在VLM微调数据中加入"多轮诊断对话"样本实现能力注入，这些样本由大模型自动合成，无需人工编写。

### 4.3 创新点三：AgriReason 农业多模态推理评测基准

#### 4.3.1 创新动机

医学领域已拥有 MedQA、PubMedQA 等标准化推理评测基准，而农业领域在多模态推理评测方面存在明显空白。现有农业数据集（PlantVillage 等）仅包含图片和标签，缺乏推理过程的标注。AgriReason 旨在填补这一空白。

#### 4.3.2 数据集构成

AgriReason 包含约 500-800 条高质量样本，每条样本包含图像、环境上下文、诊断问题、标准答案、完整推理链、难度等级和任务类型。

任务类型分布如下：

| 任务类型 | 比例 | 描述 |
|---------|------|------|
| 单一病害诊断（disease） | 35% | 标准病害识别与推理 |
| 虫害识别（pest） | 25% | 害虫种类判断与危害程度评估 |
| 鉴别诊断（differential） | 20% | 区分外观相似的病害或缺素症状 |
| 营养诊断（nutrient） | 10% | 从叶色、形态判断缺素症状 |
| 不确定性处理（uncertainty） | 10% | 图片模糊或信息不足时的合理表达 |

#### 4.3.3 自动化生成流程

AgriReason 采用全自动化生成流程，实现零人工标注：

1. 从 PlantVillage、PlantDoc、IP102、Cassava、Pest24 等公开数据集获取（图片, 标签）对。
2. 调用大模型为每张图片生成完整推理链：观察特征 - 特征指向 - 排除项 - 结论。
3. 对同一张图片运行三次推理，采用自一致性（self-consistency）策略进行质量控制——三次结果一致的纳入数据集，不一致的自动剔除。

#### 4.3.4 评测指标体系

| 指标 | 说明 |
|------|------|
| Top-1 诊断准确率 | 最终诊断与标准答案的一致性 |
| 推理链质量分数 | 与参考推理链的语义相似度（BGE-M3 计算） |
| 不确定性校准误差（ECE） | Expected Calibration Error，衡量置信度校准水平 |
| 鉴别诊断覆盖率 | 正确排除错误候选诊断的比例 |

### 5.2 数据流设计

系统数据流遵循"问诊 - 辩论 - 报告"三阶段流转：

```
用户上传图片
  |
  v
AVD模块: VLM评估信息充分度
  |
  +-- 不足 --> 生成追问指令 --> 返回前端 --> 用户补充图片 --> 循环
  |
  +-- 充分 --> 进入DDP辩论引擎
                 |
                 +-- Proposer: 分析全部图片 --> 输出诊断+证据
                 |
                 +-- Challenger: 独立分析 --> 输出质疑+替代诊断
                 |
                 +-- Proposer: 回应质疑
                 |
                 +-- Arbiter: 综合裁定 --> 最终诊断 + 推理链
                              |
                              v
                       输出生成模块
                        |  |  |
                        |  |  +-- RAG检索植保知识 --> 处置建议
                        |  +---- VLM Grounding --> 病灶标注图
                        +------- 向量检索 --> 相似病例对比
                                 |
                                 v
                           前端渲染完整报告
```

### 5.3 模块间接口规范

**AVD 到 DDP 接口**：传递包含用户提供的所有图片（1-4 张）、VLM 对每张图的结构化描述、用户输入的文字上下文以及 AVD 多轮对话历史的诊断上下文。

**DDP 到输出生成接口**：传递包含最终诊断结论、置信度等级、支持证据列表、排除诊断列表、不确定性标注、完整辩论记录及病灶标注坐标的辩论结果。

## 六、技术路线

### 6.1 多模态基座模型

**选型：Qwen2.5-VL-7B**（通义千问视觉语言模型）

选型理由：

- **国产自主**：通义千问系列为阿里巴巴自主研发，符合竞赛对国产技术的导向要求。
- **视觉定位能力**：原生支持 Visual Grounding，可输出 bounding box 坐标，实现病灶区域精准标注。
- **资源友好**：7B 参数量在单卡消费级 GPU（RTX 4090, 24GB）上即可完成 LoRA 微调。
- **中文能力**：面向中文优化，适配中文农业文献检索与报告生成。
- **开源许可**：Apache 2.0 协议，免费商用。

### 6.2 微调方案

- **方法**：QLoRA（4-bit 量化基座 + LoRA 适配器，rank=16, alpha=32）
- **框架**：PEFT + Transformers + bitsandbytes
- **梯度策略**：Gradient Checkpointing，单卡无需 DeepSpeed
- **训练数据**：约 5000-8000 条指令微调样本
- **训练时长**：单卡 RTX 4090 约 12-16 小时
- **显存占用**：约 14-18 GB（4-bit 模型约 5GB + LoRA参数 + 梯度 + 激活值）

采用 QLoRA 而非普通 LoRA 的原因：Qwen2.5-VL-7B 在 bf16 精度下约占 14GB 显存，叠加训练开销将超出 RTX 4090 的 24GB 限制。4-bit 量化将模型压缩至约 5GB，为训练过程腾出充足空间。

### 6.3 RAG 知识检索

- **向量数据库**：ChromaDB（本地运行，零成本）
- **嵌入模型**：BGE-M3（多语言，开源）
- **知识源**：《中国农作物病虫害图鉴》、植保技术手册、农药使用指南、农业院校公开课程资料等
- **检索策略**：Top-5 相关段落拼接至 Agent 提示词中，为诊断提供专业知识支撑

### 6.4 前端技术

- **框架**：React 18 + TypeScript
- **样式**：TailwindCSS
- **动画**：Framer Motion（辩论气泡动画、标注渐现效果）
- **实时通信**：WebSocket（辩论过程逐条推送至前端）
- **图表可视化**：Recharts（消融实验图表、性能雷达图）

### 6.5 后端技术

- **框架**：FastAPI + Uvicorn
- **实时推送**：WebSocket Endpoint
- **图片处理**：Pillow + OpenCV（标注渲染）
- **模型推理**：vLLM 或 Transformers Pipeline
- **部署方案**：Docker Compose（后端服务 + 模型推理 + ChromaDB）

### 6.6 技术栈与成本汇总

| 组件 | 技术选型 | 费用 |
|------|---------|------|
| 多模态基座 | Qwen2.5-VL-7B | 免费开源 |

## 七、数据方案

### 7.1 训练数据来源

全部采用免费公开数据集：

| 数据集 | 规模 | 内容 | 用途 |
|--------|------|------|------|
| PlantVillage | 54,306 张 | 38种作物病害 | VLM 微调主数据 |
| PlantDoc | 2,598 张 | 真实田间拍摄 | 增强泛化能力 |
| IP102 | 75,000+ 张 | 102种害虫 | 虫害识别训练 |
| Cassava Disease | 21,397 张 | 木薯病害 | 补充数据多样性 |
| Pest24 | 25,000+ 张 | 24种常见害虫 | 补充虫害数据 |

### 7.2 微调指令数据生成

采用全自动化流程，使用 Qwen2.5-VL-72B 生成三类指令数据：

**类型一：单轮诊断 + 推理链（约 3000 条）。** 输入为作物图片及诊断请求，输出为包含观察特征、特征分析、排除项和结论的完整推理链。

**类型二：多轮问诊对话（约 1500 条）。** 从同一病害类别中抽取 2-3 张不同角度/部位的图片组成一组，模拟"用户追问后补充拍摄"的过程。PlantDoc 等田间数据集天然包含同一病害的多角度照片，可直接配对；PlantVillage 的实验室照片则用于生成"信息充分、无需追问"的单轮负样本，训练模型判断何时不需要追问。

**类型三：辩论对话（约 1500 条）。** 按照 DDP 协议的三角色格式生成 Proposer-Challenger-Arbiter 的完整辩论记录。

生成模型优先级：首选通义千问 DashScope API 调用 Qwen2.5-VL-72B（质量最高），备选服务器本地推理 Qwen2.5-VL-7B（零成本保底）。

### 7.3 数据质量保证

- **自一致性过滤**：同一样本生成 3 次，结果一致的保留，不一致的剔除。
- **格式校验**：脚本自动检查输出是否符合预定义的 JSON Schema。
- **去重处理**：基于图像哈希 + 文本相似度去除重复样本。

## 八、实验设计

### 8.1 消融实验方案

在 AgriReason Benchmark 上进行严格的对比消融实验，验证各模块的贡献：

| 编号 | 方法 | 描述 |
|------|------|------|
| B1 | ResNet-50 | 传统图像分类基线 |
| B2 | Qwen2.5-VL (zero-shot) | 不微调直接推理 |
| B3 | Qwen2.5-VL + CoT | 零样本 + Chain-of-Thought 提示 |
| B4 | Qwen2.5-VL-FT | 微调后直接推理（无 DDP） |
| **O1** | **Qwen2.5-VL-FT + DDP** | **微调 + 辩论协议** |
| **O2** | **Qwen2.5-VL-FT + DDP + AVD** | **微调 + 辩论 + 主动问诊（完整系统）** |

### 8.2 评估维度

| 评估维度 | 说明 |
|---------|------|
| Top-1 诊断准确率 | 最终诊断与标准答案的一致性 |
| 推理链质量分数 | 与参考推理链的语义相似度 |
| 不确定性校准误差（ECE） | 置信度与实际准确率的偏差 |
| 平均推理延迟 | 系统从输入到输出的响应时间 |

### 8.3 预期结果

实验预期将验证以下递进式性能提升趋势：

**B1 << B2 < B3 < B4 < O1 < O2**

即：传统分类模型远低于视觉语言模型基线；零样本 CoT 提示优于直接推理；领域微调带来显著提升；DDP 辩论机制进一步提高准确率与可解释性；完整系统（含 AVD 主动问诊）达到最优表现。

## 十、预期成果

### 10.1 技术成果

1. **AgriMind 作物智能会诊系统**：端到端可运行的完整系统，涵盖主动问诊、多智能体辩论诊断、可视化报告生成等全流程功能。
